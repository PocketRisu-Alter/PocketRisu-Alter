# AlterRisu 디자인 시스템 지침

근거: `RESEARCH.md` · 값: `tokens.css` · 미리보기: `preview.html`

## 원칙 (우선순위 순)

1. **독서가 1순위.** 캐릭터 챗은 장문을 오래 읽는 앱이다. 대비(본문 4.5:1↑),
   행간(`--leading-body`), 줄폭(`--measure-chat` 72ch)을 어떤 장식과도 바꾸지 않는다.
2. **다크 퍼스트, 웜톤.** 순수 검정(#000)·순수 흰색(#fff) 금지. 차콜과 크림.
   라이트 테마는 다크에서 파생하며 시맨틱 토큰 이름은 동일하다.
3. **크롬은 침묵한다.** 그림자 대신 보더, 채도 높은 색은 액센트(`--accent`) 단 하나.
   화면에서 가장 눈에 띄는 것은 항상 대화 내용이어야 한다.
4. **모션은 의미가 있을 때만.** 마이크로 인터랙션(120–320ms, `--ease-out`)만 허용.
   장식용 애니메이션·글로우·그라데이션 금지. `prefers-reduced-motion` 항상 존중.
5. **상태는 3중으로.** 색만으로 상태를 전하지 말 것 — 색+모양/아이콘+텍스트.

## 레이아웃

```
┌──────────┬────────────────────────────────┐
│ sidebar  │  chat header (캐릭터 정체성)      │
│ 288px    ├────────────────────────────────┤
│          │  conversation column (≤72ch,    │
│ 캐릭터    │   가운데 정렬, 양옆 여백)          │
│ 채팅 목록  ├────────────────────────────────┤
│          │  composer (pill, ≤760px 중앙)    │
└──────────┴────────────────────────────────┘
```

- 대화 컬럼은 화면이 아무리 넓어도 `--measure-chat`을 넘지 않는다.
- 모바일(<768px): 사이드바는 오버레이로 전환, 컴포저는 풀폭.

## 컴포넌트 지침

### 메시지

- **캐릭터(AI) 메시지 = 플랫.** 말풍선·배경 없음(`--msg-char-bg: transparent`),
  본문이 문서처럼 흐른다. 메시지 시작에 캐릭터명을 `--font-display`(세리프),
  `--text-sm`, `--text-secondary`로 작게 표기.
- **사용자 메시지 = 조용한 블록.** `--msg-user-bg` + `--radius-md`, 우측 들여쓰기
  (풀폭 금지). 시각적 무게가 AI 응답보다 가벼워야 한다.
- 로르플레이 마크업: `*행동*` → `--msg-em-color` 이탤릭, `"대사"` → 본문색 유지.
- 메시지 도구(수정/삭제/재생성)는 호버/포커스 시에만 표시하되, 키보드 포커스로도
  반드시 접근 가능해야 한다.

### 스트리밍 / 백그라운드 생성 (이 제품의 간판)

- 생성 중 메시지: 좌측에 `--msg-streaming-border` 2px 보더 + 펄스 점 + "생성 중" 텍스트(3중 표시).
- 토큰 추가 시 전체 리플로우 금지 — 텍스트 노드에 append만 한다.
- **컴포저 옆에 정지(stop) 버튼 상시 노출**, 실패 시 같은 자리에 재시도.
- 다른 화면/재접속에서 돌아와도 생성 중임이 보이게: 사이드바 채팅 항목에도
  생성 중 점 표시. "창을 닫아도 계속됩니다" 안내는 `--text-muted`로 컴포저 근처에.
- 스트리밍 영역에 `aria-live="polite"` (전체 메시지가 아니라 완료 문장 단위로 갱신).

### 컴포저

- 필 형태(`--radius-pill`), 최대 `--composer-max`, 화면 하단 중앙.
- 안에는 입력창 + 전송 버튼 + (추후) "＋" 단일 메뉴만. 도구 버튼을 늘어놓지 않는다 (ChatGPT 패턴).
- 전송 버튼만 `--accent` 채움 — 화면에서 유일하게 채도 높은 면.

### 사이드바

- `--bg-raised`, 우측 `--border-subtle` 1px. 그림자 금지.
- 항목: 아바타 36px 원형 + 이름 1줄 + 보조정보 1줄(`--text-muted`).
- 선택 상태: 좌측 2px `--accent` 보더 + `--bg-hover` (색+위치 2중, 텍스트는 목록이므로 생략 허용).

### 버튼·입력

- 기본 버튼: `--bg-surface` + `--border-subtle`, 호버 시 `--bg-hover`.
- 주요 액션(전송, 저장)만 `--accent` 채움 + `--text-on-accent`.
- 파괴 액션: 평시 `--text-secondary`, 호버 시에만 `--danger` — 빨강을 상시 노출하지 않는다.
- 포커스: 모든 인터랙티브 요소에 `--focus-ring` (outline 제거 금지).
- 클릭/터치 타깃 최소 `--touch-target`(44px) — 시각 크기가 작아도 히트 영역은 패딩으로 확보.

### 다이얼로그(설정 등)

- `--bg-raised` + `--radius-lg` + `--shadow-overlay`(그림자가 허용되는 유일한 곳) + `--bg-overlay` 백드롭.
- 섹션 제목은 `--font-display` 세리프.

## 타이포그래피

| 용도 | 토큰 |
|---|---|
| 캐릭터명, 화면 제목, 빈 상태 문구 | `--font-display` (세리프) |
| 채팅 본문, UI 전반 | `--font-body` / `--text-md` / `--leading-body` / `--weight-body` |
| 코드, 토큰 수, 디버그 | `--font-mono` |

- 한국어: 세리프는 `Noto Serif KR`, 본문은 시스템 폰트 폴백 — 웹폰트를 쓰게 되면
  반드시 가변 폰트 1파일로(서브셋 포함). 본문 `word-break: keep-all` 권장.
- 다크 배경 본문은 `--weight-body`(420)와 `--leading-body`(1.7)를 따른다
  (다크에서 가는 글자는 뭉개진다 — 2026 다크모드 관행).
