# design/ — AlterRisu 디자인 시스템

UI 작업의 단일 기준점. 새 화면·컴포넌트를 만들 때 이 폴더의 규칙을 따른다.

| 파일 | 내용 |
|---|---|
| `RESEARCH.md` | 대형 LLM 업체(ChatGPT·Claude·Gemini) UI 분석 + 2026 트렌드 조사 |
| `SYSTEM.md` | 디자인 원칙·컴포넌트 지침·접근성 규칙 (지켜야 할 것) |
| `tokens.css` | 구현용 디자인 토큰 — 색·타이포·간격·모션 전부 CSS 변수로 |
| `preview.html` | 토큰이 적용된 정적 미리보기 (브라우저로 열면 됨, 서버 불필요) |

## 방향 한 줄 요약

**"따뜻한 에디토리얼 다크"** — 차가운 네온 SF가 아니라, 잘 만든 출판물 같은
조용하고 따뜻한 어두운 화면. 캐릭터 챗은 한 번에 수십 분씩 **글을 읽는** 앱이므로
독서 경험(대비·행간·줄폭)이 모든 장식보다 우선한다.

## 사용 규칙

- 색·크기·간격을 하드코딩하지 말 것. 반드시 `tokens.css`의 변수를 쓴다.
  필요한 토큰이 없으면 토큰을 추가하고 이 폴더에 기록한 뒤 사용.
- `public/style.css`를 수정할 때는 `SYSTEM.md`의 컴포넌트 지침과 대조할 것.
- 토큰을 바꾸면 `preview.html`을 브라우저로 열어 눈으로 확인.
